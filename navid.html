Roadmap Presentation

نقشه راه پروژه Mini Modular Shop

این پروژه یک API فروشگاه ماژولار با Laravel است که بر پایه‌ی معماری لایه‌ای و ساختار ماژول‌محور طراحی شده تا توسعه‌پذیر، تمیز و قابل گسترش باشد. در این صفحه، کل roadmap پروژه از راه‌اندازی اولیه تا امکانات حرفه‌ای API به‌صورت یکپارچه و مناسب ارائه نمایش داده شده است.

Roadmap پروژه

در ادامه، تمام فازهای roadmap به‌صورت ترتیبی و قابل ارائه در یک صفحه نمایش داده شده‌اند.

Phase 1

راه‌اندازی پروژه و زیرساخت

هدف این فاز، آماده‌سازی زیرساخت فنی پروژه برای توسعه ماژولار است.

کارهای اصلی

  • ایجاد پروژه Laravel
  • اتصال به MySQL
  • نصب سیستم ماژولار
  • Publish کردن تنظیمات ماژول
  • ایجاد اولین ماژول با نام Product

خروجی این فاز

Outcome: پروژه آماده توسعه ماژولار می‌شود.
تمرکز: پایه‌ریزی ساختار و استاندارد توسعه.
Phase 2

ماژول Product

در این فاز، CRUD کامل محصولات در قالب یک ماژول مستقل پیاده‌سازی می‌شود.

بخش‌های داخل ماژول

  • Controllers
  • Requests
  • Models
  • Services
  • Repositories
  • routes
  • database

API محصولات

GET/api/productsلیست محصولات
GET/api/products/{id}نمایش محصول
POST/api/productsایجاد محصول
PUT/api/products/{id}ویرایش محصول
DELETE/api/products/{id}حذف محصول
Phase 3

ماژول Category

این فاز برای دسته‌بندی محصولات و تعریف رابطه بین محصول و دسته‌بندی است.

ساختار داده

Table: categories
Fields: id, name, slug, parent_id, timestamps
Relations: Product belongsTo Category / Category hasMany Products

API دسته‌بندی

GET/api/categoriesلیست دسته‌بندی‌ها
GET/api/categories/{id}نمایش دسته‌بندی
POST/api/categoriesایجاد دسته‌بندی
PUT/api/categories/{id}ویرایش دسته‌بندی
DELETE/api/categories/{id}حذف دسته‌بندی
Phase 4

ماژول Authentication

پیاده‌سازی سیستم کاربری و احراز هویت با Laravel Sanctum.

اجزای اصلی

  • ثبت‌نام کاربر
  • ورود کاربر
  • خروج کاربر
  • مدیریت access tokenها
  • محافظت از APIهای خصوصی

API احراز هویت

POST/api/auth/registerثبت‌نام
POST/api/auth/loginورود
POST/api/auth/logoutخروج
GET/api/userاطلاعات کاربر
Phase 5

ماژول Cart

مدیریت سبد خرید برای کاربران و نگهداری آیتم‌های انتخاب‌شده.

ساختار جداول

carts: id, user_id, timestamps
cart_items: id, cart_id, product_id, quantity, price

API سبد خرید

POST/api/cart/addافزودن به سبد
GET/api/cartنمایش سبد
PUT/api/cart/updateبه‌روزرسانی سبد
DELETE/api/cart/itemحذف آیتم
Phase 6

ماژول Order

تبدیل سبد خرید به سفارش و نگهداری جزئیات سفارش‌ها.

ساختار جداول

orders: id, user_id, total_price, status, timestamps
order_items: id, order_id, product_id, price, quantity

API سفارش‌ها

POST/api/orders/checkoutثبت سفارش
GET/api/ordersلیست سفارش‌ها
GET/api/orders/{id}جزئیات سفارش
Phase 7

ماژول Payment

شبیه‌سازی فرآیند درخواست و تایید پرداخت برای درک جریان کلی پرداخت.

ویژگی این فاز

  • پرداخت به‌صورت ماک و آموزشی
  • آماده برای اتصال درگاه واقعی در آینده
  • جدا از منطق سفارش و قابل توسعه

API پرداخت

POST/api/payment/requestدرخواست پرداخت
POST/api/payment/verifyتایید پرداخت
Phase 8

بهبودهای حرفه‌ای

در این فاز، API از حالت صرفاً آموزشی به ساختاری حرفه‌ای‌تر و استانداردتر نزدیک می‌شود.

قابلیت‌های حرفه‌ای

  • API Resources مانند ProductResource و CategoryResource
  • Search روی محصولات
  • Filtering بر اساس قیمت و دسته‌بندی
  • Pagination برای لیست‌ها
  • Exception Handling اختصاصی مثل ProductNotFoundException

نمونه Queryها

GET/api/products?search=iphoneجستجو
GET/api/products?min_price=100فیلتر قیمت
GET/api/products?category=mobileفیلتر دسته‌بندی
GET/api/products?page=2صفحه‌بندی
